注解与反射

发表于 2020-06-06 更新于 2020-06-07

注解

什么是注解

注解（Annotation）是Java5.0引入的技术。

简单的说，就是程序能读懂的注释。

注解的用途：

  • 可用于向编译器提供某些指令。例如，@Override用于指示编译器这个被注解的方法是对该同名方法的重写。
  • 可以向编译器提供编译时指令，软件构建工具可以将其进一步用于生成代码，XML文件等。
  • 可以定义在运行时可用的注解，我们可以使用java反射对其进行访问，并且可以在运行时向程序提供指令。

注解的格式：“@注解名(参数)”

内置注解

  • @Override - 标记重写方法。如果发现其父类，或者是引用的接口中并没有该方法时，会报编译错误。
  • @Deprecated - 标记过时方法。表示可以用，但不建议用
  • @SuppressWarnings - 用来抑制编译时的警告信息。
  • @SafeVarargs - Java 7 开始支持，忽略任何使用参数为泛型变量的方法或构造函数调用产生的警告。
  • @FunctionalInterface - Java 8 开始支持，标识一个匿名函数或函数式接口。
  • @Repeatable - Java 8 开始支持，标识某注解可以在同一个声明上使用多次。

元注解

这个概念跟元数据很像，元数据是描述数据的数据，那么元注解就是定义注解的注解。元注解有四个，分别为：

  • @Target - 用于描述注解的适用范围

    TYPE、FIELD、METHOD、PARAMETER、CONSTRUCTOR、LOCAL_VARIABLE、ANNOTATION_TYPE、PACKAGE、TYPE_PARAMETER、TYPE_USE

  • @Retention - 用于描述注解的生命周期（即在何种级别下保留该注释信息）

    SOURCE–>CLASS–>RUNTIME

    默认为CLASS，一般建议自定义注解使用RUNTIME

  • @Document - 说明该注解将被包含在javadoc中

  • @Inherited - 说明子类可以继承父类中的该注解

反射

什么是反射

Java 反射机制在程序运行时，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性。这种 动态的获取信息 以及 动态调用对象的方法 的功能称为 java 的反射机制。反射可以被用于：

  • 在运行时分析类的能力
  • 在运行时检查对象
  • 实现泛型数组操作代码
  • 利用Method对象
  • 处理注解
  • 生成动态代理

反射机制很重要的一点就是运行时，其使得我们可以再程序运行时加载、探索及使用编译期间完全未知的.class文件。换句话说，Java 程序可以加载一个运行时才得知名称的 .class 文件，然后获悉其完整构造，并生成其对象实体、或对其 fields（变量）设值、或调用其 methods（方法）。

反射的功能十分强大和灵活，但它依旧有缺点：对性能有影响。

Class类（泛型类）

在程序运行期间，Java运行时系统始终为所有对象维护一个运行时类型标识。这个信息会跟踪每个对象所属的类，JVM利用运行时类型信息选择要执行的正确的方法。Class类可以访问这些信息。

如何获得Class类：

  • 已知具体的类，通过类的class属性获取（程序性能最高）

    一个Class对象实际上表示的是一个类型，这可能是类，也可能不是。例如int不是类，但int.class是一个Class类型的对象

  • 已知某个类的实例，调用该实例的getClass()方法

  • 已知某个类全类名，且该类在类路径下，可通过Class类的静态方法forName()获取（可能跑出ClassNotFoundException）

  • 内置基本数据类型可用类名.TYPE

  • 利用ClassLoader

    • this.getClass.getClassLoader(); // 使用当前类的ClassLoader
    • Thread.currentThread().getContextClassLoader(); // 使用当前线程的ClassLoader
    • ClassLoader.getSystemClassLoader(); // 使用系统ClassLoader，即系统的入口点所使用的ClassLoader。
